# Security Policy

## Reporting a Vulnerability

前衛小説ガイドブックでは、プロジェクトのセキュリティを真剣に受け止めています。セキュリティ上の脆弱性を発見された場合は、責任ある方法でご報告いただけると幸いです。

**GitHubのプライベート脆弱性報告機能をご利用ください。**

1.  GitHubリポジトリの「Security」タブにアクセスします。
2.  「Report a vulnerability」をクリックし、表示されるフォームに詳細を入力してください。

この機能により、脆弱性の詳細を非公開で安全に報告できます。

報告には、以下の情報を含めてください。

-   脆弱性の明確で簡潔な説明。
-   脆弱性を再現するための手順。
-   脆弱性の潜在的な影響。
-   推奨される緩和策や修正案（もしあれば）。

ご報告いただいた内容については、3営業日以内に確認の連絡を差し上げ、進捗状況についてお知らせいたします。

## Disclosure Policy

脆弱性の修正状況について、随時お知らせいたします。脆弱性が修正された際には、ご希望に応じて貢献を公開的に謝意をもって認めさせていただきます。

## General Security Best Practices

-   すべての依存関係とライブラリを最新の安定バージョンに保つ。
-   安全なコーディング慣行に従う。
-   潜在的なセキュリティ問題がないか、定期的にコードレビューを行う。
-   プロジェクトに関連するすべてのアカウントに、強力でユニークなパスワードを使用する。
-   利用可能な場合は、多要素認証（MFA）を有効にする。
